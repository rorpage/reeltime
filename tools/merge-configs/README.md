# merge-configs

Merges a Reeltime config generated by `archive_to_config.js` with one generated by `tvmaze_import.js`, combining the best fields from each source.

---

## Why

The two import tools produce complementary outputs that can't be combined in a single step:

| Field | `archive_to_config.js` | `tvmaze_import.js` |
|---|---|---|
| `url` | Real Archive.org stream URL | Blank |
| `duration` | Exact (from ffprobe) | Approximate (TVmaze runtime) |
| `description` | From Archive.org item | From TVmaze episode summary (HTML stripped) |
| `title` | Inferred from filename | Clean, from TVmaze |
| `series_title` | Not set | Set from stream name |
| `sub_title` | Not set | Episode title from TVmaze |
| `episode_num` | Inferred from filename | Authoritative from TVmaze |
| `date` | Not always set | Air date from TVmaze |
| `icon` | Archive thumbnail (whole item) | Episode image from TVmaze |

This script merges both configs into one, keying on `episode_num` (e.g. `S01E01`) to match entries from each source.

---

## Merge priorities

| Field | Winner |
|---|---|
| `url` | Archive.org |
| `duration` | Archive.org |
| `title` | TVmaze |
| `series_title` | TVmaze |
| `sub_title` | TVmaze |
| `episode_num` | TVmaze |
| `date` | TVmaze |
| `icon` | TVmaze |
| `description` | TVmaze (HTML tags already stripped) |
| `category` | TVmaze |

If the preferred source is missing a field, the other source is used as a fallback.

---

## Requirements

- Node.js 18+
- A config from `archive_to_config.js` (with real URLs)
- A config from `tvmaze_import.js` (with clean metadata)

---

## Usage

```bash
node tools/merge-configs/merge_configs.js \
  --archive brooklyn-nine-nine-archive.yaml \
  --tvmaze  brooklyn-nine-nine-tvmaze.yaml  \
  --output  brooklyn-nine-nine.yaml
```

Preview without writing:

```bash
node tools/merge-configs/merge_configs.js \
  --archive brooklyn-nine-nine-archive.yaml \
  --tvmaze  brooklyn-nine-nine-tvmaze.yaml  \
  --dry-run
```

---

## Options

| Option | Default | Description |
|---|---|---|
| `--archive <path>` | required | Config from `archive_to_config.js` |
| `--tvmaze <path>` | required | Config from `tvmaze_import.js` |
| `--output <path>` | `merged.yaml` | Output file path |
| `--dry-run` | off | Print to stdout instead of writing |

---

## Typical workflow

```bash
# 1. Generate a TVmaze scaffold for the show
node tools/tvmaze-import/tvmaze_import.js --id 49 --output brooklyn-nine-nine-tvmaze.yaml

# 2. Import episodes from Archive.org into a separate config
node tools/archive-org-import/archive_to_config.js \
  --item my-brooklyn-nine-nine-archive-item \
  --config brooklyn-nine-nine-archive.yaml

# 3. Merge the two
node tools/merge-configs/merge_configs.js \
  --archive brooklyn-nine-nine-archive.yaml \
  --tvmaze  brooklyn-nine-nine-tvmaze.yaml  \
  --output  brooklyn-nine-nine.yaml
```

---

## Notes

- **Unmatched entries** are included as-is with a warning to stderr. Archive-only entries have real URLs; TVmaze-only entries will have a blank URL with a `# TODO` comment.
- **Entries without `episode_num`** in the archive config can't be matched and are appended at the end as-is.
- The `stream:` block is taken from the TVmaze config (which has the proper show name). If the TVmaze config has no `stream:` block, the archive config's block is used.
- Re-running is safe - the output file is always fully regenerated from the two source configs.
